  <name>Task 1: SilenceChunker rolling RMS, SIMD, threshold, tail, and split fixes</name>
  <files>host/Ams.Core/Audio/SilenceChunker.cs, host/Ams.Tests/Audio/SilenceChunkerTests.cs</files>
  <action>
Four changes to SilenceChunker, all in the same file:

**1a. Rolling RMS with squared threshold in DetectSilenceRegions:**
- Before the hop loop, compute the initial sum-of-squares for samples[0..RmsWindowSize).
- Convert the linear threshold to a squared threshold: `thresholdSq = threshold * threshold`. Compare `sumOfSquares / windowSize < thresholdSq` instead of calling ComputeRms + Math.Sqrt each hop.
- On each hop step, subtract the sum-of-squares contribution of the outgoing samples (samples[pos..pos+hopSize)) and add the incoming samples (samples[pos+windowSize-hopSize..pos+windowSize)). This replaces the full-window recomputation with O(hopSize) work per step.
- Keep the existing ComputeRms method for the edge case (buffer < RmsWindowSize) but mark it clearly as edge-case-only.

**1b. SIMD ComputeRms (and rolling update helpers):**
- Add `using System.Numerics;` for `Vector<float>`.
- Create a private static helper `SumOfSquaresSimd(ReadOnlySpan<float> span)` that processes `Vector<float>.Count` elements at a time: load vector, multiply element-wise (v * v), accumulate into a `Vector<float>` sum, then horizontally sum the accumulator plus scalar remainder.
- Use this helper for the initial window computation and for adding/subtracting hop-stride slices in the rolling update.

**1c. FindChunkBoundaries tail (totalSamples % RmsHopSize):**
- After the main hop loop in DetectSilenceRegions, handle the final partial window: if `lastProcessedEnd < totalSamples` and the gap is >= some minimum (e.g., RmsHopSize/2), compute RMS for that tail slice using ComputeRms and check against threshold. This ensures silence at the very end of a buffer is not missed when `totalSamples` is not evenly divisible by RmsHopSize.
- The existing trailing-silence handler (`if silenceStart >= 0`) already covers the case where silence was *entered* before the loop ended, but it uses `totalSamples - silenceStart` which is correct. The gap is when the *last hop* ends before totalSamples and there are unexamined samples that might start a new silence region. Add a check after the loop: compute the position of the last hop end (`lastHopEnd`), and if `lastHopEnd < totalSamples`, compute RMS for `samples[lastHopEnd..totalSamples)` and handle accordingly.

**1d. SelectSplitPoints improvements:**
- Pre-size the `selected` list: `new List<int>(candidates.Count + totalLength / maxChunkSamples + 1)` to avoid reallocation.
- In the forced-split inner loop, after computing `forcedSplit`, validate that the *remaining* suffix after forcedSplit is itself partitionable: if `totalLength - forcedSplit < minChunkSamples` and `forcedSplit != totalLength`, clamp `forcedSplit = totalLength - minChunkSamples` (only if that yields `forcedSplit > lastSplit`). This prevents the forced split from emitting a tail chunk below minChunkSamples.

**Tests:**
- Add a test `TailSilence_DetectedWhenNotAlignedToHopSize` that creates a buffer whose totalSamples is not divisible by RmsHopSize, with silence in the trailing partial window. Verify the silence region is detected.
- Existing 12 SilenceChunkerTests must continue passing. The rolling RMS is a pure performance refactor -- same algorithm, same outputs.

Then for each channel, copy each source buffer's channel span into the destination at the running offset using `source.GetChannel(ch).Span.CopyTo(dest.GetChannelSpan(ch).Slice(offset, source.Length))`.

- Replace with SIMD: `using System.Numerics;`. Process `Vector<float>.Count` samples at a time. Create a `Vector<float> scaleVec = new(scale)`. For each channel, load source vectors, multiply by scaleVec, add to destination vectors. Handle scalar remainder.
- Keep the fast-path for mono (channels == 1) unchanged.

Two algorithmic fixes in the pipeline infrastructure:

**3a. MfaChunkCorpusBuilder.Build -- pre-index word-timed mapping:**
- Currently, `BuildLabTextFromWordTiming` is called per chunk (line 109) and iterates the entire `hydrate.Words` list each time checking if each word's ASR token midpoint falls within the chunk's time range. With C chunks and W words, this is O(C * W).
- Pre-index before the chunk loop: create a sorted structure mapping words to their ASR token midpoint times. Since chunks are non-overlapping and ordered by StartSec, use a single pass:
  1. Before the chunk loop, build a `List<(double MidpointSec, HydratedWord Word)>` for all words that have a valid `AsrIdx` within `asr.Tokens` bounds. Sort by MidpointSec.
  2. Replace the per-chunk `BuildLabTextFromWordTiming(hydrate.Words, asr, ...)` call with a new private method `BuildLabTextFromPreIndexedWords(sortedWordMidpoints, chunk.StartSec, chunk.EndSec)` that uses binary search (`List<T>.BinarySearch` with a custom comparer, or manual lower-bound via span) to find the start index in the sorted list, then scans forward until midpoint >= chunkEndSec + tolerance.
  3. This yields O(W log W) upfront sort + O(W) total scanning across all chunks = O(W log W + C) instead of O(C * W).
- Keep the existing `BuildLabTextFromWordTiming` method with its `internal static` signature untouched for test compatibility. The optimization lives only in `Build()`.

**3b. FfFilterGraphRunner accumulator channel count -- derive from sink output:**
- In `FilterGraphExecutor` constructor (line 112), `_channels` is set from `_inputs[0].Channels`. When filters change channel topology (e.g., pan=mono on stereo input), the accumulator gets the wrong channel count.
- In `RefreshOutputFormat()`, after reading the sink's sample rate, also read the sink's negotiated channel count. Use the FFmpeg API: `ffmpeg.av_buffersink_get_channels(_sink)` returns int. If positive, store in a new field `_outputChannels`.
- Add field `private int _outputChannels;` initialized to `_inputs[0].Channels` in constructor (as fallback).
- In `RefreshOutputFormat()`, after the sample rate query, add: `var sinkChannels = ffmpeg.av_buffersink_get_channels(_sink); if (sinkChannels > 0) _outputChannels = sinkChannels;`
- Move the `_accumulator` initialization to AFTER `RefreshOutputFormat()` completes (it is currently before). Use `_outputChannels` instead of `_channels` for the accumulator and frameSink initialization.
- Keep `_channels` (rename consideration is optional) for input-side usage in SendFrame interleaving.
