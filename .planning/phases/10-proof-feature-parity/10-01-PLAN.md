**Validation-Viewer Parity Required:**
The Python validation-viewer provides these API endpoints that must be replicated:
- `/api/chapters` - List chapters with metrics (sentenceCount, sentenceFlagged, sentenceAvgWer, paragraphCount, paragraphFlagged)
- `/api/overview` - Book-wide stats (chapterCount, totalSentences, totalFlaggedSentences, avgSentenceWer, totalParagraphs, totalFlaggedParagraphs)

**Data Source:**
All metrics come from HydratedTranscript which contains:
- Sentences with: Id, BookText, ScriptText, Status, Metrics (Wer, Cer), Timing, Diff
- Paragraphs with: Id, BookRange, SentenceIds, BookText, Metrics (Wer, Cer, Coverage), Status, Diff

**ChapterMetrics record:**
- SentenceCount, SentenceFlagged, SentenceAvgWer (formatted as "2.48%")
- ParagraphCount, ParagraphFlagged, ParagraphAvgWer

**BookOverview record:**
- BookName, ChapterCount
- TotalSentences, TotalFlaggedSentences, AvgSentenceWer
- TotalParagraphs, TotalFlaggedParagraphs, AvgParagraphWer
- Chapters (list of ChapterInfo with name and metrics)

**Flagging logic:**
- Sentence is flagged if Status != "ok" OR Diff != null
- Paragraph is flagged if any of its sentences are flagged
- WER averaging: weighted by sentence/paragraph count

```csharp
[ApiController]
[Route("api/proof")]
public class ProofApiController : ControllerBase
{
    // GET /api/proof/chapters - List all chapters with metrics
    [HttpGet("chapters")]
    public ActionResult<IEnumerable<ChapterInfo>> GetChapters();

    // GET /api/proof/overview - Book-wide aggregate metrics
    [HttpGet("overview")]
    public ActionResult<BookOverview> GetOverview();

    // GET /api/proof/report/{chapterName} - Detailed chapter report
    [HttpGet("report/{chapterName}")]
    public ActionResult<ChapterReport> GetReport(string chapterName);
}
```

**Implementation:**
- Inject BlazorWorkspace, ValidationMetricsService, ProofReportService
- GetChapters: Iterate AvailableChapters, open each via workspace, compute metrics
- GetOverview: Call GetChapters, aggregate with ValidationMetricsService.ComputeBookOverview
- GetReport: Open chapter via workspace, get HydratedTranscript, build report via ProofReportService

**Error handling:**
- Return 404 if chapter not found
- Return 400 if workspace not initialized

**Performance note:**
GetChapters/GetOverview need to load all hydrate files - this may be slow for large books.
Consider caching metrics in BlazorWorkspace.SetWorkingDirectory for future optimization.
