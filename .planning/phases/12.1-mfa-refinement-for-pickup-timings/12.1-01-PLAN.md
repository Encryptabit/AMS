<objective>
Run pickup audio through MFA forced alignment to replace rough ASR token boundaries with phoneme-accurate sentence timings.

Purpose: ASR (Whisper) gives word-level timestamps that are approximate. MFA forced alignment uses the acoustic model with known text to produce sub-phoneme-accurate boundaries. For pickup replacements, precise sentence boundaries mean cleaner crossfade splices with no clipped words or excess silence.

Output: PickupMfaRefinementService that runs MFA on matched pickup audio, and integration into PickupMatchingService so all returned PickupMatch records have MFA-refined timings.
</objective>

  <action>
Create `PickupMfaRefinementService` in the Workstation.Server Services namespace. This service takes a list of ASR-matched `PickupMatch` records, the pickup WAV path, and the corresponding `HydratedSentence` list, then runs MFA forced alignment to produce phoneme-accurate sentence boundaries.

**Constructor dependencies:** `BlazorWorkspace` (for working directory).

**Public method:** `RefineWithMfaAsync(string pickupFilePath, IReadOnlyList<PickupMatch> asrMatches, IReadOnlyList<HydratedSentence> flaggedSentences, CancellationToken ct) -> Task<IReadOnlyList<PickupMatch>>`

**Implementation flow:**

1. **Check MFA cache** -- Use same hash strategy as ASR cache (SHA256 of fullpath|size|lastwrite) but with `.mfa.json` suffix stored at `{workDir}/.polish/pickups/{hash}.mfa.json`. If cache hit, deserialize and return directly.

2. **Build temp corpus directory** at `{workDir}/.polish/pickups/mfa/{hash}/`:
   - Copy pickup WAV as `pickup.wav` (use `File.Copy`)
   - Write `pickup.lab` containing the concatenated BookText of ALL matched sentences (in match order by PickupStartSec), normalized through `PronunciationHelper.ExtractPronunciationParts()` and joined with spaces. Use the same PrepareLabLine pattern from MfaWorkflow: for each sentence's BookText, call `PronunciationHelper.ExtractPronunciationParts(bookText)`, join parts with space, then join all sentence lines with newline into the .lab file content. Actually -- MFA .lab files are a single block of text (not per-line sentences), so join ALL sentence parts into one continuous line separated by spaces.

3. **Ensure MFA supervisor is warm** -- call `MfaProcessSupervisor.EnsureReadyAsync(ct)`.

4. **Build MfaChapterContext:**
   ```
   CorpusDirectory = corpusDir (the temp dir with pickup.wav + pickup.lab)
   OutputDirectory = Path.Combine(corpusDir, "output")
   WorkingDirectory = corpusDir
   DictionaryModel = MfaService.DefaultDictionaryModel
   AcousticModel = MfaService.DefaultAcousticModel
   Beam = 80
   RetryBeam = 200
   SingleSpeaker = true
   CleanOutput = true
   ```

5. **Run MFA validate + G2P + align** -- Follow the same OOV handling pattern as MfaWorkflow.RunChapterAsync but simplified (no corpus fallback needed since pickup text is known-good from ASR match):
   - Create `MfaService(useDedicatedProcess: false)` directly (same as MfaWorkflow does)
   - Run validate (optional, skip if no OOV file found)
   - If OOVs found, run G2P + AddWords to create custom dictionary
   - Run `service.AlignAsync(context, ct)` -- this produces `{outputDir}/pickup.TextGrid` (or `{outputDir}/alignment/mfa/pickup.TextGrid` depending on MFA version)

6. **Parse TextGrid** -- Find the TextGrid file (check both candidate paths like MfaWorkflow does), call `TextGridParser.ParseWordIntervals(textGridPath)` to get `IReadOnlyList<TextGridInterval>`.

7. **Map word intervals to sentences** -- This is the core mapping step. Strategy:
   - For each matched sentence, we know its BookText. Normalize the BookText words using `PronunciationHelper.ExtractPronunciationParts()`.
   - Build a flat list of all expected words across all matched sentences (in order), tracking which sentence each word belongs to.
   - Use the MfaTimingMerger's alignment approach: convert TextGrid intervals to `TextGridWord` records, build `SentenceTarget` records for each matched sentence.
   - BUT -- since we don't have book indices here (this is pickup-local, not chapter-level), use a simpler approach:
     - Build a flat sequence of expected normalized words from all matched sentences
     - Align TextGrid word tokens against this flat sequence using Needleman-Wunsch (reuse MfaTimingMerger.MergeAndApply with synthetic bookIdx values)
     - OR simpler: sequential greedy mapping. Walk TextGrid words left-to-right. For each sentence, consume words until the sentence's expected word count is exhausted. Track the earliest Start and latest End of consumed words as that sentence's MFA timing.
   - **Recommended approach (simplest, most robust):** Sequential sentence-word mapping:
     - `sentenceWordCounts[i]` = number of normalized words in sentence i's BookText (via PronunciationHelper)
     - Walk TextGrid intervals sequentially, filtering out empty/silence entries
     - For each sentence in order, assign the next N TextGrid word intervals where N = sentenceWordCounts[i]
     - Sentence i's MFA start = first assigned interval's Start, MFA end = last assigned interval's End
     - If there are more TG words than expected (MFA may split contractions differently), use a fuzzy consume strategy: keep consuming while the next TG word's normalized form matches expected words

   Actually, use the Needleman-Wunsch approach from MfaTimingMerger since it handles count mismatches robustly. Here's how:
   - Assign synthetic "bookIdx" values: sentence 0 words get indices 0..N-1, sentence 1 words get indices N..N+M-1, etc.
   - Track sentenceRanges: `{sentenceId, startIdx, endIdx}` mapping each sentence to its synthetic index range
   - Create `TextGridWord[]` from parsed intervals (filter empty text)
   - Create `SentenceTarget[]` from sentenceRanges where SetTiming captures (start, end, duration)
   - Create a getBookToken function from the flat expected word list
   - Call `MfaTimingMerger.MergeAndApply(textGridWords, getBookToken, 0, totalWords-1, wordTargets, sentenceTargets)`
   - Read back each sentence's timing from the SentenceTarget callbacks

8. **Build refined PickupMatch list** -- For each original ASR match, if MFA produced a timing for that sentence, create a new PickupMatch with the MFA timings. If MFA failed for a specific sentence, keep the original ASR timing. Preserve the original Confidence and RecognizedText.

9. **Cache results** -- Serialize the refined match list to `{hash}.mfa.json`.

10. **Cleanup** -- Delete the temp corpus directory (try-catch, don't fail if cleanup fails).

**Error handling:** Wrap the entire MFA flow in try-catch. If MFA fails at any stage (supervisor not available, alignment fails, TextGrid missing), log a warning and return the original ASR matches unchanged. This ensures the pickup import workflow never breaks even if MFA is unavailable.

**Important implementation details:**
- `MfaProcessSupervisor.EnsureReadyAsync` and `MfaService` are `internal` -- the Workstation.Server project already references Ams.Core. Check if InternalsVisibleTo is set. If not, use the public surface: `MfaProcessSupervisor.TriggerBackgroundWarmup()` is internal too. Actually, looking at MfaWorkflow, it creates `new MfaService(useDedicatedProcess: false)` which is `internal` constructor. Since MfaWorkflow is itself `internal` to Ams.Core, the workstation server can't call it directly. **Resolution:** Add `[assembly: InternalsVisibleTo("Ams.Workstation.Server")]` to Ams.Core if not already present, OR create a thin public API on MfaService. Check for existing InternalsVisibleTo first. If it exists, use the internal APIs directly. If not, add the attribute to a suitable file in Ams.Core (e.g., create `Properties/AssemblyAttributes.cs` or add to an existing one).
- Actually, MfaWorkflow.PrepareLabLine is `private static`. We don't need it -- just use `PronunciationHelper.ExtractPronunciationParts()` directly (which is what PrepareLabLine calls). PronunciationHelper is public.
- MfaTimingMerger.MergeAndApply is public. TextGridParser.ParseWordIntervals is public. These can be called directly.
