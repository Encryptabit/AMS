Create `PickupAssetService.cs` as a transient service (stateless orchestrator, like PolishService).

**Core API:**
- `ImportAsync(string sourcePath, IReadOnlyList<CrxPickupTarget> crxTargets, CancellationToken ct)` — auto-detects source type:
  - If `sourcePath` is a directory → `ImportFromFolderAsync`
  - If `sourcePath` is a single WAV file → `ImportFromSessionFileAsync`
  - Returns `(IReadOnlyList<PickupAsset> Matched, IReadOnlyList<PickupAsset> Unmatched)` tuple

- `ImportFromFolderAsync(string folderPath, IReadOnlyList<CrxPickupTarget> crxTargets, CancellationToken ct)`:
  1. Get all `*.wav` files in the folder, sorted alphabetically
  2. For each file, extract error number from filename using `TryExtractErrorNumber()` — supports patterns: `NNN.wav` (direct), `error_NNN.wav`/`err_NNN.wav` (prefix), trailing digits fallback
  3. If error number matched → find corresponding CRX target by ErrorNumber
  4. Run ASR on the file for `TranscribedText`
  5. Compute confidence via `LevenshteinMetrics.Similarity(normalized_asr, normalized_should_be)` using existing `NormalizeForMatch` pattern (lowercase, remove punctuation, collapse whitespace)
  6. Create `PickupAsset` with `SourceType = IndividualFile`, `TrimStartSec = 0`, `TrimEndSec = file_duration`
  7. Assets with `MatchedErrorNumber == null` or `Confidence < 0.2` go to Unmatched bucket

- `ImportFromSessionFileAsync(string sessionFilePath, IReadOnlyList<CrxPickupTarget> crxTargets, CancellationToken ct)`:
  1. Run the enhanced matching pipeline (delegates to PickupMatchingService)
  2. Wrap each matched result in a `PickupAsset` with `SourceType = SessionSegment`
  3. Low-confidence matches (< 0.3) go to Unmatched bucket

**Cache logic:**
- Cache file: `{workDir}/.polish/pickup-assets-cache.json`
- Cache key: `PickupAssetCache` with `SourceFilePath + SourceFileSizeBytes + SourceFileModifiedUtc + CrxTargetsFingerprint`
- On import: check cache. If cached entry exists with matching key triple + CRX fingerprint, return cached assets immediately.
- CRX fingerprint: SHA256 of sorted `ErrorNumber:SentenceId` pairs joined by `;`
- If cache miss or stale: process and save new cache entry.

**DI dependencies:** `PickupMatchingService`, `BlazorWorkspace` (for workDir). Register as `builder.Services.AddTransient<PickupAssetService>()` in Program.cs.

**Text normalization:** Reuse the existing `NormalizeForMatch` pattern from PickupMatchingService (lowercase, remove punctuation, collapse whitespace). Make it a shared static method or call it directly.

Enhance the existing PickupMatchingService with two changes:

**1. Add `MatchByTextSimilarity` method** for out-of-order session file matching:

This is a new public method that replaces pure positional matching when the user's recording order doesn't match CRX order:

```csharp
public static List<PickupMatch> MatchByTextSimilarity(
    IReadOnlyList<PickupSegment> segments,
    IReadOnlyList<CrxPickupTarget> targets)
```

Algorithm (greedy best-first assignment per research Pattern 7):
1. For each segment, compute `LevenshteinMetrics.Similarity(NormalizeForMatch(segment.TranscribedText), NormalizeForMatch(target.ShouldBeText))` against ALL remaining targets
2. Build a score matrix: segments × targets
3. Pick the highest-confidence pair, assign it, remove both from the available pools, repeat
4. Stop when best remaining score < 0.2 (no useful matches left)
5. Remaining unmatched segments get `PickupMatch` with `Confidence = 0.0` and `IsLowConfidence = true`

This is O(n² · m) which is negligible for typical pool sizes (5-30 items).

**2. Enhance segmentation with MFA-aware gap threshold:**

In the existing `SegmentUtterances` method (or wherever silence gap-based segmentation occurs), add a parameter or overload:
- When MFA-refined timings are available, use a lower gap threshold of 0.4s instead of the current 0.8s
- MFA phone boundaries give more precise word endpoints, so shorter silences between them are legitimate utterance breaks
- Add a `double utteranceGapThresholdSec = 0.8` parameter with default maintaining backward compatibility
- The caller (PickupAssetService) passes `0.4` when MFA refinement was applied

**3. Make `NormalizeForMatch` public static** if it isn't already, so PickupAssetService can call it for confidence scoring.

Do NOT remove the existing `PairSegmentsToTargets` positional matching — it's still used as a fallback. The new `MatchByTextSimilarity` is the primary path for session files; positional matching is kept for cases where ASR fails (zero transcribed text).

Strategy selection in matching pipeline:
- If segments have transcribed text → use `MatchByTextSimilarity`
- If ASR failed for most segments → fall back to `PairSegmentsToTargets` positional matching
