<objective>
Enhance SpliceBoundaryService with breath-aware boundary placement. When a proposed cut point bisects a detected breath, the cut is shifted to avoid splitting the breath — keeping it whole with whichever sentence it perceptually belongs to.

Purpose: The current silence-center approach can place cuts through breaths, causing audible artifacts at splice points. Breath-aware placement ensures cleaner, more natural-sounding edits.
Output: Enhanced SpliceBoundaryService with a `RefineBoundariesBreathAware` public method and a `BreathAware` enum value added to `BoundaryMethod`.
</objective>

**2. Add `RefineBoundariesBreathAware` public static method** — a new entry point that calls the existing `RefineBoundaries` first, then checks for breath overlap at each refined boundary:

```csharp
public static SpliceBoundaryResult RefineBoundariesBreathAware(
    AudioBuffer chapterBuffer,
    double roughStartSec,
    double roughEndSec,
    double? prevSentenceEndSec,
    double? nextSentenceStartSec,
    SpliceBoundaryOptions? options = null)
```

Implementation flow:
1. Call existing `RefineBoundaries()` to get the initial refinement (silence-center or snap-energy).
2. For the **start boundary**: define a search region `[refinedStart - 0.2, refinedStart + 0.2]` clamped to buffer bounds. Call `FeatureExtraction.Detect(chapterBuffer, searchStart, searchEnd, new FrameBreathDetectorOptions())` to find breaths in this narrow window.
3. If any breath straddles the proposed start cut point (breath.StartSec <= refinedStart <= breath.EndSec), shift the cut to **after the breath** (`breath.EndSec + 0.005`). This keeps the breath with the preceding sentence. Set `startMethod = BoundaryMethod.BreathAware`.
4. Do the same for the **end boundary**: search `[refinedEnd - 0.2, refinedEnd + 0.2]`. If a breath straddles the end cut, shift to **before the breath** (`breath.StartSec - 0.005`). This keeps the breath with the following sentence. Set `endMethod = BoundaryMethod.BreathAware`.
5. Re-apply the existing `ApplyBoundaryPadding` step to the breath-adjusted positions (same padding logic).
6. Final sanity check: if `refinedStart >= refinedEnd`, fall back to originals (same as existing fallback).

**Important:** Only run breath detection in the inter-sentence gap region, NOT inside the speech content. The 0.2s search window around each boundary is appropriate because breaths naturally occur at sentence transitions. Use default `FrameBreathDetectorOptions()` which already has `FricativeGuardMs = 25` to prevent false positives on fricative consonants (f, s, sh, th).

**Note on FeatureExtraction.Detect():** This method accepts `AudioBuffer`, `startSec`, `endSec`, and `FrameBreathDetectorOptions`. It returns `IReadOnlyList<Region>` where each Region has `StartSec` and `EndSec` properties (relative to the start of the buffer, not the search region). Verify the exact API signature by reading the file before implementing.

Do NOT modify the existing `RefineBoundaries` method — the new `RefineBoundariesBreathAware` wraps it.
