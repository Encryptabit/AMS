Read the current PolishService.cs (952 lines). This is a major refactor — work carefully.

**Add new core method — `RebuildChapterAsync`:**
```csharp
public async Task<AudioBuffer> RebuildChapterAsync(
    ChapterContextHandle handle,
    IReadOnlyList<ChapterEdit> editsToApply,
    CancellationToken ct)
```

Implementation (per research Pattern 2):
1. Start from the original treated audio: `handle.Chapter.Audio.Treated?.Buffer` (throw if null)
2. Sort edits by `BaselineStartSec` DESCENDING (back-to-front) — this is critical! Each edit modifies content after it, so applying from the end preserves upstream positions.
3. For each edit in sorted order:
   - Load the replacement audio via `UndoService.LoadReplacementSegment(chapterStem, edit.Id)`
   - Apply the appropriate AudioSpliceService operation based on `edit.Operation`:
     - `PickupReplace` → `AudioSpliceService.ReplaceSegment(buffer, edit.BaselineStartSec, edit.BaselineEndSec, replacement, edit.CrossfadeDurationSec, edit.CrossfadeCurve)`
     - `RoomtoneInsert` → `AudioSpliceService.InsertAtPoint(buffer, edit.BaselineStartSec, replacement, edit.CrossfadeDurationSec, edit.CrossfadeCurve)`
     - `RoomtoneReplace` → `AudioSpliceService.ReplaceSegment(buffer, edit.BaselineStartSec, edit.BaselineEndSec, replacement, edit.CrossfadeDurationSec, edit.CrossfadeCurve)`
     - `RoomtoneDelete` → `AudioSpliceService.DeleteRegion(buffer, edit.BaselineStartSec, edit.BaselineEndSec, edit.CrossfadeDurationSec, edit.CrossfadeCurve)`
4. Return the rebuilt buffer

**Refactor `ApplyReplacementAsync`:**
- Before the splice, save the pickup replacement audio via `UndoService.SaveReplacementSegmentAsync` (new from Plan 04)
- Still save original segment backup via existing `UndoService.SaveOriginalSegmentAsync`
- Create `ChapterEdit` and add to `EditListService` (this may already be done by StagingQueueService — verify and avoid duplicate)
- After the edit is recorded, call `RebuildChapterAsync` with ALL current edits to produce the chapter audio
- Persist the rebuilt buffer as `corrected.wav`

**Refactor `RevertReplacementAsync`:**
- Remove the edit from `EditListService` (may already be done by StagingQueueService — coordinate)
- Get remaining edits from `EditListService.GetEdits(chapterStem)`
- If remaining edits exist: call `RebuildChapterAsync` with remaining edits
- If no remaining edits: restore to original treated audio (copy treated buffer as corrected)
- Persist result as `corrected.wav`

**Remove:**
- `RebaseTranscriptTimeToCurrentTimeline` — replace all calls with `TimelineProjection.BaselineToCurrentTime(time, editListService.GetEdits(chapterStem))`
- Any surgical splice-back revert logic
- Any direct ShiftDownstream calls (should already be gone from Plan 04, but clean up any remnants in PolishService)

**Refactor `ApplyRoomtoneOperationAsync`:**
- Convert roomtone operations to `ChapterEdit` records (use `EditOperation.RoomtoneInsert/Replace/Delete`)
- Save replacement audio (the roomtone fill or null for delete) via UndoService
- Add to EditListService
- Rebuild chapter from all edits

**Inject new dependencies:** `EditListService`, `PickupAssetService` (from Plan 03). Keep existing dependencies.

**1. Replace `SpliceBoundaryService.RefineBoundaries` calls with `RefineBoundariesBreathAware`:**

Wherever PolishService computes splice boundaries (typically in the staging/apply flow when determining where to cut the chapter audio), change:
```csharp
// OLD
var boundaries = SpliceBoundaryService.RefineBoundaries(buffer, start, end, prevEnd, nextStart);
// NEW
var boundaries = SpliceBoundaryService.RefineBoundariesBreathAware(buffer, start, end, prevEnd, nextStart);
```

**2. Content-aware handle sizing (replace PickupSlicePaddingSec):**

The current code uses `PickupSlicePaddingSec = 0.080` (80ms) for padding around the splice region. This is too small — the crossfade (70ms) consumes nearly all of it.

Replace with content-aware handle sizing:
- After breath-aware boundary refinement, the refined boundaries already account for breath positions
- The handle size should be at least `crossfadeDuration + 30ms` beyond the speech edge in both directions
- For a default 70ms crossfade: handles need at least 100ms of non-speech audio on each side
- Use the breath detection result to determine where speech actually ends/begins, then add `crossfadeDuration + BreathGuardSec` padding

Remove or deprecate the `PickupSlicePaddingSec` constant. The boundary service now handles proper placement.

**3. Update `GeneratePreviewAsync` / audition method** for context playback:

When the user auditions a staged replacement, generate a preview that includes surrounding chapter audio:
- Extract `contextWindowSec` (default 2.0s) of chapter audio before the replacement region
- Extract `contextWindowSec` of chapter audio after the replacement region  
- Splice: `[context_before] + [pickup_audio] + [context_after]` with crossfade at each join
- Play the spliced preview

This uses existing `AudioSpliceService.ReplaceSegment` — build a context buffer from the treated audio by trimming `[replacement.OriginalStartSec - contextWindow, replacement.OriginalEndSec + contextWindow]`, then replacing the center with the pickup.
