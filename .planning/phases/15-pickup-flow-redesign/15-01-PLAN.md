public enum EditOperation { PickupReplace, RoomtoneInsert, RoomtoneReplace, RoomtoneDelete }
public enum PickupSourceType { SessionSegment, IndividualFile }

public sealed record ChapterEdit(
    string Id,
    string ChapterStem,
    EditOperation Operation,
    double BaselineStartSec,      // position in ORIGINAL (baseline) timeline
    double BaselineEndSec,        // position in ORIGINAL (baseline) timeline
    double ReplacementDurationSec, // duration of what was inserted (0 for delete)
    int? SentenceId,              // null for roomtone ops

    string? PickupAssetId,        // reference to PickupAsset used
    double CrossfadeDurationSec,
    string CrossfadeCurve,
    DateTime AppliedAtUtc);

public sealed record PickupAsset(
    string Id,                      // GUID
    PickupSourceType SourceType,
    string SourceFilePath,          // original WAV path
    double TrimStartSec,            // within the source file
    double TrimEndSec,              // within the source file
    string TranscribedText,         // ASR result
    double Confidence,              // match confidence 0.0-1.0
    int? MatchedErrorNumber,        // CRX error number if matched
    int? MatchedSentenceId,         // target sentence if matched
    string? MatchedChapterStem,     // target chapter if matched
    DateTime ImportedAtUtc);

    string SourceFilePath,

**2. Create TimelineProjection.cs** in `host/Ams.Core/Audio/`:

A pure static class (following AudioSpliceService pattern — no DI needed). Two public methods:

- `BaselineToCurrentTime(double baselineTimeSec, IReadOnlyList<ChapterEdit> appliedEdits)` — walks the edit list front-to-back. For each edit: if edit is fully upstream of the query time, shift by delta (`ReplacementDurationSec - (BaselineEndSec - BaselineStartSec)`). If query time falls inside the edit region, clamp to edit boundary. If edit is downstream, skip. Return `Math.Max(0, current)`.

- `ProjectedDuration(double baselineDurationSec, IReadOnlyList<ChapterEdit> appliedEdits)` — delegates to `BaselineToCurrentTime(baselineDurationSec, appliedEdits)`.

The `ChapterEdit` type lives in the Workstation Models namespace. TimelineProjection is in Ams.Core, so it needs a reference. Since Ams.Core should not reference Ams.Workstation.Server, use a generic interface approach: make the method accept `IReadOnlyList<T>` with a simple record interface, OR simply place the `ChapterEdit` record in Ams.Core (in `Ams.Core/Audio/` namespace) since it's a pure data type with no UI dependencies. **Recommended:** Define `ChapterEdit` and `EditOperation` in `Ams.Core/Audio/` as they are domain models, not UI models. Keep `PickupAsset`/`PickupAssetCache`/`PickupSourceType` in Workstation Models since they're import-workflow specific.

Update the namespace in PolishModels.cs to reference the Core types via `using Ams.Core.Audio;` if needed, or add the Core types to a new file `host/Ams.Core/Audio/ChapterEditModels.cs`.

Create `EditListService.cs` as a singleton service (follows StagingQueueService pattern — singleton + lazy-load per chapter + JSON persistence).

**Storage:** Persist to `{workDir}/.polish/edit-list.json` — same `.polish/` directory used by StagingQueueService.

**API surface:**
- `Add(ChapterEdit edit)` — appends to the list, persists immediately
- `Remove(string editId)` — removes by ID, persists immediately  
- `GetEdits(string chapterStem)` — returns `IReadOnlyList<ChapterEdit>` for a chapter, sorted by `AppliedAtUtc`
- `GetAllEdits()` — returns all edits across all chapters
- `Clear(string chapterStem)` — removes all edits for a chapter
- `HasEdits(string chapterStem)` — quick check

**Serialization:** Use `System.Text.Json` with `JsonSerializerOptions { WriteIndented = true }`. Serialize `IReadOnlyList<ChapterEdit>` directly. Use a simple `lock` for thread safety (single-user app, same pattern as StagingQueueService).

**Lazy loading:** Load from disk on first access. Cache in `Dictionary<string, List<ChapterEdit>>` keyed by chapter stem. Save the entire dictionary on every mutation (same as StagingQueueService pattern).

**DI registration:** Register as `builder.Services.AddSingleton<EditListService>()` in Program.cs. The service needs `BlazorWorkspace` injected to resolve `workDir`.

**Migration note:** This service co-exists with the existing `StagingQueueService` during the transition. Later plans will remove `ShiftDownstream` from `StagingQueueService` and have it delegate to `EditListService`. For now, `EditListService` is standalone.
