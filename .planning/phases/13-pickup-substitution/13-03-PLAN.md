    ```csharp
    private string? _roomtoneFilePath;
    private bool _isProcessing;
    private string? _processingStatus;
    private double _processingProgress; // 0.0-1.0

    // Cross-chapter pickup data
    private Dictionary<string, List<CrossChapterPickupMatch>> _matchesByChapter = new();

    // Flipper navigation
    private List<ChapterCrxInfo> _chaptersWithCrx = new();
    private int _currentChapterIndex = -1;

    // Current chapter state
    private WaveformPlayer? _waveformPlayer;
    private long _correctedVersion;
    private IReadOnlyList<HydratedSentence>? _flaggedSentences;
    private IReadOnlyList<StagedReplacement>? _stagedItems;

    private record ChapterCrxInfo(string ChapterName, string Stem, int CrxCount, bool AllCommitted);
    ```

    **Layout (top to bottom):**

    1. **Header Bar** — BitStack Horizontal with AutoHeight:
       - BitTextField for pickup session file path + "Set" BitButton
       - BitTextField for roomtone file path + "Set" BitButton
       - When _isProcessing: BitProgress (linear, not circular) below with _processingStatus text

    2. **Row 1 — Breadcrumb** — BitStack Horizontal with BitText segments:
       - Format: `{ChapterName} ({chapterPos}/{totalChapters}) — {committed}/{crxCount} committed`
       - Use BitTag for completion badge when all committed (BitColor.Success, "Complete")

    3. **Row 2 — Waveform with Flippers** — BitStack Horizontal:
       - Left: BitButton with `<` icon (BitIconName.ChevronLeft), disabled if !CanGoPrev
       - Center: WaveformPlayer component (flex: 1), AudioUrl from GetAudioUrl(), full chapter waveform
       - Right: BitButton with `>` icon (BitIconName.ChevronRight), disabled if !CanGoNext

    4. **Row 3 — Three-Column Pipeline** — BitGrid Columns="12":
       - Column 1 (ColumnSpan 4): "Matches" header + list of PickupBox components for matched items
       - Column 2 (ColumnSpan 4): "Staged" header + list of PickupBox components for staged items
       - Column 3 (ColumnSpan 4): "Committed" header + list of PickupBox components for committed items
       - Each column has a header with count badge

    **Flipper Navigation Logic:**
    - OnInitialized: Pre-compute _chaptersWithCrx by filtering Workspace.AvailableChapters for those with CRX entries (same ChapterMatches logic from existing ChapterPolish.razor)
    - If no chapters with CRX, show info message
    - NavigateToChapter(int index): update _currentChapterIndex, call Workspace.SelectChapter, reload flagged sentences and staging queue, reload waveform
    - CanGoNext / CanGoPrev computed properties

    **Column State Derivation (per Research Pattern 1):**
    - GetMatchedItems(): _matchesByChapter[currentChapter] filtered to exclude items that have a StagedReplacement (Staged or Applied status)
    - GetStagedItems(): _stagedItems where Status == Staged
    - GetCommittedItems(): _stagedItems where Status == Applied

    **Audio URL pattern:** Same as ChapterPolish.razor: `/api/audio/chapter/{chapterName}/corrected?v={_correctedVersion}`

    Use BitUI components throughout (BitStack with AutoHeight, BitCard, BitTag, BitButton, BitText, BitProgress, BitGrid, BitGridItem). Reference .planning/references/bit-blazorui-components.md via grep for exact API if needed.

    **Parameters:**
    ```csharp
    [Parameter] public PickupMatch? Match { get; set; }
    [Parameter] public StagedReplacement? Replacement { get; set; }
    [Parameter] public PickupBoxState State { get; set; }
    [Parameter] public string? PickupFilePath { get; set; }
    [Parameter] public HydratedSentence? Sentence { get; set; }
    [Parameter] public EventCallback<PickupMatch> OnStage { get; set; }
    [Parameter] public EventCallback<string> OnUnstage { get; set; }
    [Parameter] public EventCallback<string> OnCommit { get; set; }
    [Parameter] public EventCallback<string> OnRevert { get; set; }
    [Parameter] public EventCallback<string> OnAudition { get; set; }
    ```

    **Layout (BitCard wrapper):**
    1. Mini waveform canvas (200x40px) — `<canvas>` element with unique id
    2. Matched sentence text (truncated to ~80 chars) from Sentence?.BookText
    3. Confidence badge (BitTag with color: green >= 0.8, yellow >= 0.5, red below) — only shown when Match is not null
    4. Time range text (caption): `{PickupStartSec:F1}s - {PickupEndSec:F1}s`
    5. Action buttons based on State:
       - Matched: "Stage" BitButton (calls OnStage)
       - Staged: "Unstage" (calls OnUnstage), "Audition" (calls OnAudition)
