  <name>Task 2: PickupImporter component</name>
  <files>host/Ams.Workstation.Server/Components/Shared/PickupImporter.razor</files>
  <action>
Create `PickupImporter.razor` in `Components/Shared/`:

**Parameters:**
- `[Parameter] public string ChapterStem { get; set; }` -- current chapter
- `[Parameter] public IReadOnlyList<HydratedSentence>? FlaggedSentences { get; set; }` -- sentences with CRX errors
- `[Parameter] public EventCallback<List<PickupMatch>> OnMatchesReady { get; set; }` -- notify parent of matches
- `[Parameter] public EventCallback<StagedReplacement> OnStageRequested { get; set; }` -- request staging a match

**State:**
- `_pickupFilePath` (string?) -- selected pickup file
- `_matches` (List<PickupMatch>?) -- ASR match results
- `_isProcessing` (bool) -- loading state
- `_errorMessage` (string?) -- error display

**UI Layout** (use Bit components per MEMORY.md):
1. **File selection**: `BitTextField` for file path + `BitButton` "Browse" (for now, text input -- Blazor Server can't do native file picker, user pastes path)
   - Also show a "Use File" button that triggers import
2. **Processing state**: `BitProgress` with "Matching pickups..." when processing
3. **Match results** (when _matches populated):
   - `BitStack Vertical` with one `BitCard` per match
   - Each card shows: recognized text, matched sentence excerpt, confidence badge (`BitTag` -- green for >0.8, yellow for 0.5-0.8), pickup time range
   - "Stage" button on each card -- creates StagedReplacement via OnStageRequested
4. **Pickup waveform**: When a match is selected, show a small `<WaveformPlayer>` of the pickup file with an editable region showing the match boundaries
   - Use `WaveformPlayer.AddEditableRegion()` for boundary adjustment
   - On region update, refresh the match's PickupStartSec/PickupEndSec

Inject `PickupMatchingService` for the matching logic. Call `MatchPickupAsync` on import.

Handle errors gracefully: file not found, ASR failure, no matches found.

  <action>
Create `StagingQueue.razor` in `Components/Shared/`:

**Parameters:**
- `[Parameter] public string ChapterStem { get; set; }`
- `[Parameter] public IReadOnlyList<StagedReplacement>? Items { get; set; }`
- `[Parameter] public EventCallback<string> OnApply { get; set; }` -- apply by replacement ID
- `[Parameter] public EventCallback<string> OnRevert { get; set; }` -- revert by replacement ID
- `[Parameter] public EventCallback<string> OnRemove { get; set; }` -- unstage by replacement ID
- `[Parameter] public EventCallback OnApplyAll { get; set; }` -- apply all staged
- `[Parameter] public EventCallback<string> OnPreview { get; set; }` -- preview play a staged item

**UI Layout** (right-panel style per research recommendation):
- Header: "Staging Queue" title + item count badge + "Apply All" button (disabled if no Staged items)
- Empty state: "No replacements staged" message
- Item list: `BitStack Vertical Gap="0.5rem"` with `BitCard` per item
  - Each card shows:
    - Sentence ID + brief excerpt text (first 50 chars of recognized text)
    - Time range: `{OriginalStartSec:F1}s - {OriginalEndSec:F1}s`
    - Status badge: `BitTag` colored by status:
      - Staged = BitColor.Info (blue)
      - Applied = BitColor.Success (green)
      - Reverted = BitColor.Warning (orange)
      - Failed = BitColor.Error (red)
    - Action buttons row:
      - If Staged: "Preview" (play icon), "Apply" (checkmark), "Remove" (X icon)
      - If Applied: "Preview" (play icon), "Revert" (undo icon)
      - If Reverted: "Re-stage" or "Remove"
    - Crossfade info: small text showing `{CrossfadeDurationSec*1000:F0}ms {CrossfadeCurve}` crossfade

Use `BitButton Variant="BitVariant.Text"` for action buttons (compact). Use `BitIcon` for visual cues.

No DI injection needed -- this is a pure display component receiving data via parameters and firing EventCallbacks up to the parent ChapterPolish page.
