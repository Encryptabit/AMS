  <files>host/Ams.Workstation.Server/wwwroot/js/waveform-interop.js</files>
  <action>
Add the following new exported functions to `waveform-interop.js`, preserving all existing functions unchanged:

**`addEditableRegion(elementId, id, start, end, color, dotNetRef)`**:
- Gets wavesurfer instance and regionsPlugin (auto-init regions if not yet initialized)
- Calls `regionsPlugin.addRegion({ id, start, end, color: color || 'rgba(59, 200, 120, 0.3)', drag: true, resize: true, minLength: 0.1 })`
- Listens to region `update-end` event (NOT `update` -- per research Pitfall 6, avoid continuous events)
- On `update-end`: calls `dotNetRef.invokeMethodAsync('OnRegionUpdated', id, region.start, region.end)` with error catch
- Returns the region object for chaining

**`updateRegionBounds(elementId, regionId, start, end)`**:
- Finds region by ID, updates start/end programmatically
- For manual correction from .NET side

**`getRegionBounds(elementId, regionId)`**:
- Returns `{ start, end }` for a region, or null if not found

**`syncPlayheads(elementIds, timeSeconds)`**:
- Iterates all element IDs in the array
- For each, gets the wavesurfer instance
- Calls `wavesurfer.seekTo(timeSeconds / duration)` where duration = `wavesurfer.getDuration()`
- Skip instances where duration is 0 (not loaded)

**`playSegment(elementId, startSec, endSec)`**:
- Seeks to startSec: `wavesurfer.seekTo(startSec / duration)`
- Plays: `wavesurfer.play()`
- Sets up a one-time audioprocess listener that pauses when currentTime >= endSec
- Returns void (fire-and-forget from JS side)

**`setZoom(elementId, pxPerSec)`**:
- Calls `wavesurfer.zoom(pxPerSec)` to change zoom level
- Needed for multi-waveform view to show different zoom levels per chapter

Do NOT modify any existing functions. All new functions are appended to the module.

**New Parameters:**
```csharp
/// <summary>
/// Callback fired when a draggable region's boundaries are updated by the user.
/// Receives (regionId, newStart, newEnd).
/// </summary>
[Parameter]
public EventCallback<(string Id, double Start, double End)> OnRegionUpdated { get; set; }
```

**New Public Methods:**

`AddEditableRegion(string id, double start, double end, string? color = null)`:
- Calls `_module.InvokeVoidAsync("addEditableRegion", _elementId, id, start, end, color, _dotNetRef)`
- This enables the pickup boundary adjustment UI

`UpdateRegionBounds(string id, double start, double end)`:
- Calls `_module.InvokeVoidAsync("updateRegionBounds", _elementId, id, start, end)`

`PlaySegment(double startSec, double endSec)`:
- Calls `_module.InvokeVoidAsync("playSegment", _elementId, startSec, endSec)`
- For listen-with-context and segment preview

`SetZoom(int pxPerSec)`:
- Calls `_module.InvokeVoidAsync("setZoom", _elementId, pxPerSec)`

**New JSInvokable callback:**
```csharp
[JSInvokable]
public async Task OnRegionUpdated(string regionId, double start, double end)
{
    await OnRegionUpdated.InvokeAsync((regionId, start, end));
}
```

Note: The `OnRegionUpdated` EventCallback and the JSInvokable method have the same name but different signatures -- the EventCallback is a parameter (property), the JSInvokable is a method. If the compiler complains about ambiguity, rename the JSInvokable to `OnRegionBoundsUpdated` and update the JS `invokeMethodAsync` call accordingly.

Do NOT change any existing parameters, methods, or markup. Only add new capabilities.
