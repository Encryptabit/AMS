**AudioController -- Add region endpoint:**

Add a new endpoint to `AudioController`:
```csharp
[HttpGet("chapter/{chapterName}/region")]
public IActionResult GetChapterRegionAudio(string chapterName, [FromQuery] double start, [FromQuery] double end)
```
- Validate: start >= 0, end > start, end <= duration
- Decode the chapter's treated audio file path from the workspace (use `_workspace.GetChapterAudioPath(chapterName)` or equivalent -- check how existing GetChapterAudio resolves the path)
- Use `AudioProcessor.Decode(path, new AudioDecodeOptions(Start: TimeSpan.FromSeconds(start), Duration: TimeSpan.FromSeconds(end - start)))` for partial decode
- Encode to WAV stream: `buffer.ToWavStream()`
- Return `File(stream, "audio/wav", enableRangeProcessing: true)`
- If the chapter audio is already loaded in AudioBufferContext, use `AudioProcessor.Trim` on the in-memory buffer instead (faster)

**MultiWaveformView.razor:**

Create in `Components/Shared/`. This is the DAW-style stacked view.

Parameters:
- `[Parameter] public IReadOnlyList<ChapterWaveformConfig> Chapters { get; set; }` -- list of chapters to display
- `[Parameter] public EventCallback<double> OnTimeChanged { get; set; }` -- synchronized time update

Internal model (define in @code block):
```csharp
public record ChapterWaveformConfig(
    string ChapterName,
    string AudioUrl,
    double RegionStartSec,  // start of loaded region
    double RegionEndSec,    // end of loaded region
    int Height = 80);
```

Layout:
- `BitStack Vertical Gap="0.25rem" AutoHeight="true"` containing one `WaveformPlayer` per chapter
- Each waveform has reduced height (80px) since multiple are stacked
- Chapter name label above each waveform
- Synchronized playhead: when any waveform fires `TimeUpdated`, call `syncPlayheads` JS interop on all other instances
- Store all waveform element IDs in a list for sync calls

Playhead synchronization logic:
- Track `_waveformRefs` dictionary (chapterName -> WaveformPlayer reference)
- On any `TimeUpdated(double time)`, convert to absolute time (regionStart + time) and sync others
- Use debouncing: only sync if time delta > 0.05s from last sync (avoid feedback loops)

Memory efficiency: Each waveform loads via the region endpoint URL:
`/api/audio/chapter/{name}/region?start={regionStart}&end={regionEnd}`

Per research: load region = flagged sentence range + 30s padding on each side.
