Create `StagingQueueService` as a singleton service in `Ams.Workstation.Server.Services` namespace. Follow the pattern from `ReviewedStatusService` (singleton, persistence via JSON, keyed by book).

Internal state: `Dictionary<string, List<StagedReplacement>>` keyed by chapter stem.

Public methods:
- `Stage(StagedReplacement item)` -- adds to queue for the item's chapter
- `Unstage(string replacementId)` -- removes by ID (only if status is Staged)
- `GetQueue(string chapterStem)` -- returns `IReadOnlyList<StagedReplacement>` for a chapter
- `GetAllQueued()` -- returns all items across chapters with status == Staged
- `UpdateStatus(string replacementId, ReplacementStatus newStatus)` -- transitions status (e.g., Staged -> Applied, Applied -> Reverted)
- `Clear(string chapterStem)` -- clears all staged items for a chapter
- `ClearAll()` -- clears everything

Persistence: Save to `{workDir}/.polish/staging-queue.json` on every mutation. Load on first access (lazy). Use `BlazorWorkspace` to get the work directory path. Serialize with `System.Text.Json` using `JsonSerializerOptions { WriteIndented = true }`.

Thread safety: use `lock` on all mutations (single-user app, simple synchronization).

Constructor takes `BlazorWorkspace` for path resolution.

<task type="auto">
  <name>Task 2: UndoService and DI registration</name>
  <files>host/Ams.Workstation.Server/Services/UndoService.cs, host/Ams.Workstation.Server/Program.cs</files>
  <action>
Create `UndoService` as a singleton in `Ams.Workstation.Server.Services`.

**Undo storage strategy** (per research recommendation): Versioned segment files on disk.

Internal state: `Dictionary<string, List<UndoRecord>>` keyed by chapter stem, loaded from manifest.

Public methods:
- `SaveOriginalSegment(string chapterStem, int sentenceId, AudioBuffer originalBuffer, double startSec, double endSec, double replacementDurationSec)` -- trims the segment from `startSec` to `endSec`, encodes as WAV to `.polish-undo/{chapterStem}/sent{sentenceId}.v{N}.original.wav`, creates UndoRecord, saves manifest.
- `GetUndoRecords(string chapterStem)` -- returns `IReadOnlyList<UndoRecord>`
- `GetUndoRecord(string replacementId)` -- returns single record or null
- `LoadOriginalSegment(string replacementId)` -- decodes the backup WAV back to AudioBuffer
- `RemoveRecord(string replacementId)` -- deletes backup file and record
- `HasUndo(string replacementId)` -- checks if undo is available

**Manifest persistence**: `.polish-undo/{chapterStem}/manifest.json` -- JSON array of UndoRecord. Load on first access per chapter. Save on every mutation.

**Version numbering**: For the same sentenceId, increment version: `sent5.v1.original.wav`, `sent5.v2.original.wav`, etc. Scan existing files to determine next version.

Constructor takes `BlazorWorkspace`. Use `AudioProcessor.Trim` for segment extraction and `AudioProcessor.EncodeWav` for saving, `AudioProcessor.Decode` for loading.

**DI Registration** in `Program.cs`: Add after the CrxService line:
```csharp
// Polish area services
builder.Services.AddSingleton<StagingQueueService>();
builder.Services.AddSingleton<UndoService>();
```
