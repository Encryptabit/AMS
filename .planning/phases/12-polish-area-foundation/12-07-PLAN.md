  <action>
Create `PolishVerificationService` in `Ams.Workstation.Server.Services`.

Constructor DI: `BlazorWorkspace`, `ReviewedStatusService`, `StagingQueueService`.

**Public methods:**

`RevalidateSegmentAsync(AudioBuffer chapterBuffer, double startSec, double endSec, string expectedText, CancellationToken ct)`:
1. Trim the affected segment: `AudioProcessor.Trim(chapterBuffer, TimeSpan.FromSeconds(startSec), TimeSpan.FromSeconds(endSec))`
2. Prepare for ASR: `AsrAudioPreparer.PrepareForAsr(segment)`
3. Run ASR: `await AsrProcessor.TranscribeBufferAsync(asrReady, null, ct)`
4. Extract recognized text from ASR response
5. Compute WER/similarity: `LevenshteinMetrics.Similarity(expectedText, recognizedText)`
6. Return `RevalidationResult` record:
   ```csharp
   public sealed record RevalidationResult(
       string RecognizedText,
       string ExpectedText,
       double Similarity,      // 0.0-1.0
       bool Passed,            // similarity >= threshold (0.9)
       IReadOnlyList<AsrToken> Tokens);
   ```

`SyncToProofAsync(string chapterStem, int sentenceId, bool passed)`:
- If `passed`: mark the sentence's CRX error as resolved in the Proof area
- Update via `ReviewedStatusService` -- add the sentence to a "fixed" status
- This provides the auto-sync: "when a fix passes re-validation and user is satisfied, sentence status automatically updates in the Proof area"
- Note: The actual Proof UI will pick up the status change on next load since ReviewedStatusService is singleton and persistent

`GetRevalidationHistory(string chapterStem)`:
- Returns list of recent re-validations for the chapter (in-memory cache, cleared on chapter change)

Register as `builder.Services.AddTransient<PolishVerificationService>()` in Program.cs.

Define the `RevalidationResult` record in the same file (below the class).

**ContextPlayer.razor:**

Create a compact verification player in `Components/Shared/` for listen-with-context.

Parameters:
- `[Parameter] public WaveformPlayer? WaveformRef { get; set; }` -- reference to the chapter waveform
- `[Parameter] public double SegmentStartSec { get; set; }`
- `[Parameter] public double SegmentEndSec { get; set; }`

UI Layout:
1. **Play controls**:
   - "Play with Context" button -- plays from `(SegmentStart - ContextSeconds)` to `(SegmentEnd + ContextSeconds)` via `WaveformRef.PlaySegment()`

2. **Validation results** (if ValidationResult is set):
   - Similarity score badge: green if Passed, red if not

   - WER display
3. **Decision buttons**:
   - "Accept Fix" (green) -- fires OnAccept, syncs to Proof
