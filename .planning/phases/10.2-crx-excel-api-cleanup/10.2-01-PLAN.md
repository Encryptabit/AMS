  <action>
1. Add ClosedXML NuGet package to the .csproj:
   ```xml
   <PackageReference Include="ClosedXML" Version="0.104.2" />
   ```
   ClosedXML is MIT-licensed, the most popular .NET OSS Excel library, and has an API analogous to Python's openpyxl. Use the latest stable 0.104.x version. Do NOT use EPPlus (commercial license for v5+).

2. Rewrite `CrxService.cs` to use Excel instead of JSON. Keep the same public API surface (`Submit`, `GetEntries`, `FormatTimecode`) so CrxModal.razor requires zero changes.

   **Template copy logic (matching Python `shutil.copy2`):**
   - Template source: `C:\Aethon\BASE_CRX.xlsx` (hardcoded constant, same as Python `CRX_TEMPLATE_PATH`)
   - On first CRX submit, if the target `.xlsx` does not exist, copy the template using `File.Copy(source, dest)` which preserves formatting/styles
   - Target path: `{workspace.RootPath}/CRX/{BookName}_CRX.xlsx` (replacing the old `_CRX.json` path)
   - If template file is missing at source, throw a clear error: "CRX template not found at C:\Aethon\BASE_CRX.xlsx"

   **Excel write logic (matching Python validation-viewer exactly):**
   - Open the workbook with `new XLWorkbook(excelPath)`
   - Get the first worksheet (the template has one sheet)
   - Data starts at row 11 (`CRX_DATA_ROW_START = 11`)
   - Target row = `11 + (errorNumber - 1)` where errorNumber comes from `ExportResult.ErrorNumber` (WAV file counter from AudioExportService - DO NOT CHANGE AudioExportService)
   - Column mapping (1-indexed):
     - Column B (2): Error number formatted as 3-digit string `$"{errorNumber:D3}"` (e.g., "001", "042")
     - Column C (3): Recording Day - leave blank (same as Python)
     - Column D (4): Chapter name string
     - Column E (5): PDF/Word Page # - leave blank (same as Python)
     - Column F (6): Timecode in `HH:MM:SS` format (reuse existing `FormatTimecode` method)
     - Column G (7): Error type string (MR, PRON, DIC, etc.)
     - Column H (8): Comments string (the "Should be: / Read as:" text)
   - Save the workbook after writing: `workbook.Save()`
   - Dispose the workbook properly (use `using` statement)

   **GetEntries (read from Excel):**
   - Read the .xlsx file instead of JSON
   - Open workbook, iterate rows starting at 11
   - Stop when column B is empty (no more entries)
   - Map columns back to CrxEntry records
   - For fields not stored in Excel (SentenceId, StartTime, EndTime, AudioFile, CreatedAt), use default values (0, 0.0, 0.0, "", DateTime.MinValue) since these are only needed for the JSON format and are not used by any UI after submit
   - If the .xlsx file doesn't exist, return empty list (same behavior as JSON path)

   **Remove all JSON-related code:**
   - Remove `GetCrxJsonPath` method
   - Remove `System.Text.Json` using
   - Add `using ClosedXML.Excel;`

   **IMPORTANT:** Do NOT modify AudioExportService.cs. The WAV file counter logic there is correct and drives the error numbering. CrxService receives the error number from the ExportResult returned by AudioExportService.
