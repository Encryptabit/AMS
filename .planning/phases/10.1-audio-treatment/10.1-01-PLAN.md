- Roomtone from book's `roomtone.wav` file (not extracted per-chapter)

<task type="auto">
  <name>Task 1: Create AudioTreatmentService</name>
  <files>host/Ams.Core/Audio/AudioTreatmentService.cs, host/Ams.Core/Audio/TreatmentOptions.cs</files>
  <action>
Create AudioTreatmentService in Ams.Core/Audio with:

1. **TreatmentOptions record** with configurable durations:
   - PrerollSeconds = 0.75
   - ChapterToContentGapSeconds = 1.5
   - PostrollSeconds = 3.0
   - SilenceThresholdDb = -50.0 (for detection)

2. **TreatChapterAsync method** that:
   a. Loads chapter audio via AudioProcessor.Decode()
   b. Loads roomtone.wav from book directory
   c. Detects speech boundaries using AudioProcessor.DetectSilence() to find:
      - Title start: first speech onset
      - Title end: first significant silence gap (>1s) after title
      - Content start: speech onset after title gap
      - Content end: final speech offset
   d. Extracts segments using AudioProcessor.Trim():
      - Title segment: from title start to title end
      - Content segment: from content start to content end
   e. Assembles final audio by concatenating:
      - Roomtone trimmed to PrerollSeconds
      - Title segment
      - Roomtone trimmed to ChapterToContentGapSeconds
      - Content segment
      - Roomtone trimmed to PostrollSeconds
   f. Writes to {chapter}.treated.wav via AudioProcessor.EncodeWav()

3. **Use FFmpeg concat demuxer pattern** for assembly:
   - Create temp files for each segment
   - Use ffmpeg concat demuxer (file-based, not filter) for seamless joins
   - Clean up temp files after

**Important:** The title/content boundary detection is the key challenge. The hydrate.json first sentence marks the title, but its timing may be unreliable. Use silence detection to find the actual gap between title read and content start (typically 1-2 seconds of silence).

Do NOT use FfFilterGraph.Custom() with concat filter - it's complex for multi-input. Use the simpler file-based concat demuxer approach.

Create TreatCommand following existing CLI patterns (see DspCommand, ValidateCommand):

1. **Command structure:** `treat [chapters]`
   - chapters argument: "0,1,2" or "all" (same pattern as other commands)
   - --work-dir option for output directory (defaults to chapter artifact dir)
   - --roomtone option for explicit roomtone.wav path (defaults to book dir)

   - --force to overwrite existing treated.wav

2. **Handler implementation:**
   a. Resolve workspace and book context
   b. Parse chapter selection (single, comma-separated, or "all")
   c. For each chapter:
      - Open chapter context via workspace.OpenChapter()

      - Call AudioTreatmentService.TreatChapterAsync()
      - Report progress

3. **Register command** in Program.cs root command

4. **Error handling:**
   - Missing roomtone.wav: clear error message with expected path
   - Missing hydrate.json: skip chapter with warning
   - Audio processing failure: log and continue to next chapter

Follow the existing logging pattern using Serilog (Log.Info, Log.Debug, Log.Error).
