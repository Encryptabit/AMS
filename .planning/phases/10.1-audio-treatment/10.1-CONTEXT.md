# Phase 10.1: Audio Treatment - Context

**Gathered:** 2026-02-02
**Status:** Ready for planning

<vision>
## How This Should Work

Process chapter audio to create production-ready `treated.wav` files with proper spacing:

```
[0.75s roomtone] → [chapter title read] → [1.5s roomtone] → [chapter content] → [3.0s roomtone]
```

The command fits the existing CLI pattern: `treat 0,1,2|all` to process individual chapters or the entire book in batch.

The chapter title read is already part of the source audio (first sentence in hydrate.json). However, the timing in hydrate.json shows 0s start when the actual audio begins later - FFmpeg silence detection determines the true boundaries.

Roomtone comes from a shared `roomtone.wav` file per book, not extracted per-chapter.

</vision>

<essential>
## What Must Be Nailed

- **Correct timing assembly** - Getting the chapter title → gap → content boundaries right using FFmpeg silence detection
- **Proper roomtone insertion** - Using the book's roomtone.wav for all padding (not silence)
- **Batch capability** - Process all chapters with single command

</essential>

<boundaries>
## What's Out of Scope

- Audio normalization/loudness (no LUFS adjustments)
- Multiple output formats (WAV only, no MP3/M4B conversion)
- UI integration (CLI only, Blazor workstation later)
- Roomtone extraction (assumes roomtone.wav already exists)

</boundaries>

<specifics>
## Specific Ideas

- Output: `{chapter}.treated.wav`
- Uses existing FFmpeg integration for silence detection and audio assembly
- Timing durations: 0.75s preroll, 1.5s chapter-to-content gap, 3.0s postroll
- Chapter selection follows existing `0,1,2,3|all` paradigm

</specifics>

<notes>
## Additional Context

This is a detour inserted mid-Phase 10 to add pipeline functionality before continuing with Blazor workstation feature parity.

The hydrate.json contains sentence timings, with the first sentence being the chapter title read. The start time is unreliable (shows 0s) so actual boundaries come from FFmpeg silence detection.

</notes>
