**Prior decisions affecting this phase:**
- AudioTreatmentService uses FFmpeg silence detection and segment extraction (Phase 10.1)
- AudioBuffer stores planar float arrays with Channels/SampleRate/Length properties
- FfFilterGraph provides in-memory audio processing without temp files
- AudioProcessor.Trim() uses FfFilterGraph internally for in-memory trimming
- AudioProcessor.EncodeWav() writes final WAV to disk

**Current implementation inefficiencies (from 10.1-01-SUMMARY):**
1. External FFmpeg process - spawns ffmpeg.exe for concat; should use internal operations
2. Temp file artifacts - creates intermediate WAV files for each segment
3. Multiple encodes - encodes segments to temp files then concat copies

**Optimization targets:**
- Remove external FFmpeg process spawning
- Remove temp file artifacts (no intermediate WAV files)
- In-memory buffer concatenation
- Single encode to disk at the end

  <files>host/Ams.Core/Audio/AudioTreatmentService.cs</files>
  <action>
Refactor `TreatChapterCoreAsync` to eliminate temp files and external FFmpeg:

1. **Remove temp directory creation** - no more `Path.Combine(Path.GetTempPath(), "ams", "treat", ...)`

2. **Keep segments as AudioBuffer objects** instead of encoding to files:
   - prerollBuffer = PrepareRoomtoneSegment(...)  // already returns AudioBuffer
   - titleBuffer = AudioProcessor.Trim(...)       // already returns AudioBuffer
   - gapBuffer = PrepareRoomtoneSegment(...)      // already returns AudioBuffer
   - contentBuffer = AudioProcessor.Trim(...)     // already returns AudioBuffer
   - postrollBuffer = PrepareRoomtoneSegment(...) // already returns AudioBuffer

3. **Replace ConcatAudioFilesAsync with in-memory concat**:
   - Build list of AudioBuffer segments (preroll, title?, gap?, content, postroll)
   - Call `AudioBuffer.Concat(segments.ToArray())`
   - Single call to `AudioProcessor.EncodeWav(outputPath, concatenatedBuffer)`

4. **Remove the ConcatAudioFilesAsync private method entirely** - no longer needed

5. **Remove try/finally temp directory cleanup** - no temp files to clean

6. **Keep async signature** for API compatibility, but method becomes synchronous internally (wrap with Task.FromResult if needed, or convert to sync method called TreatChapter)

Note: PrepareRoomtoneSegment already returns AudioBuffer, so no changes needed there.
